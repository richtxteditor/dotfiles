## Session Start
1. Read tasks/lessons.md — apply all lessons before touching anything
2. Read tasks/todo.md — understand current state
3. If neither exists, create them before starting

## Tools
- Use fff for file search and grep within git-indexed directories

## Workflow
- Plan mode for complex tasks (5+ files touched or architectural decisions)
- Write plans to tasks/todo.md before implementing
- If something goes wrong: STOP, re-plan — never push through
- Use subagents for independent subtasks; one task per subagent
- After any correction: append to tasks/lessons.md
  Format: [date] | what went wrong | rule to prevent it

## Principles
- Minimal changes — only touch what's needed
- Root causes only — no workarounds or temp fixes
- Never assume — verify paths, APIs, variables before using
- Ask once upfront if unclear, then work autonomously
